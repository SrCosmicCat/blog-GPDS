Análisis de seguridad

Escrito por: Miguel Ángel Diosdado Rodríguez y Fernanda Montserrat Hernández Ávila
Fecha: 28 de octubre de 2024

Introducción

En esta entrada se presenta el análisis de seguridad realizado a la página web del proyecto SIMAPE.

Este infome se realizó con el software ZAP, una herramienta de código abierto que permite encontrar vulnerabilidades en aplicaciones web.

Resultado del análisis

El análisis de seguridad realizado a la página web del proyecto SIMAPE arrojó los siguientes resultados:

Aquí se muestra que no se tienen vulnerabilidades graves o críticas, por lo que el software es medianamente seguro. En la siguiente tabla se muestra cuáles son las vulnerabilidades a las que está expuesto el sistema:

  • Auscencia de tokens Anti-CSRF: Se encontró que la página no cuenta con tokens anti CSRF, lo que puede permitir a un atacante realizar peticiones en nombre del usuario.
  • Cabecera Content Security Policy: Se encontró que la página no cuenta con una cabecera content security policy, lo que puede permitir a un atacante realizar ataques de tipo clickjacking.
  • Configuración incorrecta Cross-Domain: Se encontró que la página no cuenta con una configuración correcta de Cross-Domain, lo que puede permitir a un atacante realizar ataques de tipo CORS.
  • Falta de cabecera Anti-Clickjacking: Se encontró que la página no cuenta con una cabecera anti-clickjacking, lo que puede permitir a un atacante realizar ataques de tipo clickjacking.
  • Cookie sin el atributo Same-Site: Se encontró que la página no cuenta con el atributo Same-Site en las cookies, lo que puede permitir a un atacante realizar ataques de tipo CSRF.
  • Encabezado 'X-Powered-By': Se encontró que la página cuenta con el encabezado 'X-Powered-By', lo que puede permitir a un atacante conocer la tecnología utilizada en el servidor.
  • X-Content-Type-Options: Se encontró que la página no cuenta con la cabecera X-Content-Type-Options, lo que puede permitir a un atacante realizar ataques de tipo MIME sniffing.

Recomendaciones

Para mejorar la seguridad de la página web del proyecto SIMAPE, se recomienda:

  • Implementar tokens Anti-CSRF en las peticiones: Se recomienda implementar tokens anti CSRF para evitar que un atacante realice peticiones en nombre del usuario.
  • Configurar una cabecera Content Security Policy: Se recomienda configurar una cabecera content security policy para evitar ataques de tipo clickjacking.
  • Configurar correctamente el Cross-Domain: Se recomienda configurar correctamente el Cross-Domain para evitar ataques de tipo CORS.
  • Implementar una cabecera Anti-Clickjacking: Se recomienda implementar una cabecera anti-clickjacking para evitar ataques de tipo clickjacking.
  • Agregar el atributo Same-Site: Se recomienda agregar el atributo Same-Site en las cookies para evitar ataques de tipo CSRF.
  • Eliminar el encabezado 'X-Powered-By': Para evitar que un atacante conozca la tecnología utilizada en el servidor.
  • Agregar la cabecera X-Content-Type-Options: Para evitar ataques de tipo MIME sniffing.

Conclusiones

El análisis de seguridad realizado a la página web del proyecto SIMAPE permitió identificar las vulnerabilidades a las que está expuesto el sistema y las recomendaciones para mejorar su seguridad.

Es importante que se tomen en cuenta las recomendaciones mencionadas para evitar posibles ataques y proteger la información de los usuarios.
